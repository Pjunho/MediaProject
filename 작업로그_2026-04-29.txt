[1단계] 코드 점검 및 버그 수정
-------------------------------------------------------

■ 코인 즉시 지급 타이밍 불일치 수정 (GameManager.cs)

  발견 내용:
  - AllyPlacer.cs의 OnReachedGoal 콜백에서 FloatingText "+1 코인"이
    아군이 골에 도달하는 즉시 표시되었으나, 실제 currentCoins는
    웨이브 종료 시 AwardWaveCoins()에서 일괄 증가하는 방식이었음.
  - 기획서에는 "아군이 골에 도착하는 즉시 코인이 지급된다"고 명시되어 있어
    UI 피드백과 실제 코인 지급 시점이 불일치하는 버그.

  수정 내용:
  - GameManager.ReportGoal()에 currentCoins++ 및 UpdateCoinHUD() 추가 →
    아군 골 도달 즉시 코인 1개 지급 및 HUD 갱신.
  - CheckWaveCompletion()의 두 웨이브 클리어 경로(최종/비최종)에서
    AwardWaveCoins() 호출 제거.
  - AwardWaveCoins() 메서드 전체 삭제.

  영향 범위: GameManager.cs (ReportGoal, CheckWaveCompletion)


[2단계] 신규 기능 구현 — 스킬/업그레이드 실패 시 토스트 알림
-------------------------------------------------------

■ 구현 내용 (GameManager.cs)

  배경:
  - 스킬 해금 버튼(AllyOrderPanel)을 눌렀을 때 코인이 부족하거나,
    업그레이드 버튼을 눌렀을 때 코인 부족 또는 최고 레벨인 경우
    기존에는 아무런 UI 피드백 없이 조용히 실패했음.
  - 플레이어 입장에서 버튼이 왜 작동하지 않는지 알 수 없는 UX 문제.

  구현 세부:
  1. BuildToastUI(Transform parent) 메서드 추가
     - HUD 캔버스에 화면 중앙 하단(y=-200) 위치에 310×48 반투명 패널 생성.
     - 내부에 Text 컴포넌트(fontSize=20, Bold) 생성.
     - 초기 비활성 상태(SetActive(false)).

  2. ShowToast(string message, Color color) public 메서드 추가
     - 중복 호출 시 기존 코루틴 중단 후 재시작.
     - ToastCoroutine() 호출.

  3. ToastCoroutine(string, Color) IEnumerator 추가
     - 1.1초 동안 완전 불투명 유지 → 이후 0.7초에 걸쳐 페이드 아웃.
     - Time.unscaledDeltaTime 사용 (일시정지 상태에서도 표시).
     - 완료 후 토스트 오브젝트 비활성화.

  4. TryUnlockSkill() 수정
     - 해금 실패 시 (!SkillSystem.IsUnlocked 조건) →
       "코인이 부족합니다!" 빨간색 토스트 표시.

  5. TryUpgrade() 수정
     - 최고 레벨(GetNextCost < 0) → "이미 최고 레벨입니다!" 황금색 토스트.
     - 코인 부족 → "코인이 부족합니다!" 빨간색 토스트.

  수정 파일: GameManager.cs
