1단계: 코드 점검 및 버그 발견

■ StageSelect.cs — OnLockedStage() 논리 버그 발견
  - 문제: 잠긴 스테이지 클릭 시 호출되는 OnLockedStage(int si)에서
    `int prevStars = StageManager.GetSavedStars(si - 1)` 로 이전 스테이지의
    별 수를 확인하여 두 가지 메시지를 분기하고 있었음.
  - 원인: 스테이지 si가 잠겨 있다는 것은 이미 GetSavedStars(si-1) == 0임을
    의미하므로(IsStageUnlocked는 stars > 0 일 때만 true), prevStars > 0 조건은
    항상 false가 되어 "별 1개 이상으로 클리어하면 해금됩니다" 메시지가
    절대 표시되지 않는 dead code 상태였음.
  - 수정: `IsStageUnlocked(si - 1)` 로 조건 변경:
      · si-1이 접근 가능한 상태(unlock)라면 → "별 1개 이상으로 클리어하면 해금됩니다"
      · si-1 자체도 잠겨 있다면 → "STAGE si-1을 먼저 클리어해야 합니다"
    이제 논리적으로 두 메시지가 모두 올바르게 표시됨.

2단계: 신규 기능 구현 — 아군 체력바 표시
----------------------------------------------------

■ 배경
  - 아군(AllyBase)이 적에게 피해를 받을 수 있는 구조이지만 (TakeDamage, HpRatio 등
    관련 코드 이미 존재), 플레이어가 아군의 현재 체력 상태를 시각적으로 확인할
    방법이 없었음 (피격 시 빨간 깜빡임만 존재).
  - 아군 체력바를 추가하면 전략적 판단(누가 위험한지)에 도움이 됨.

■ 구현 내용 (AllyBase.cs)
  필드 추가:
    · private GameObject     hpBarRoot  — 체력바 루트 오브젝트
    · private SpriteRenderer hpBarFill  — 채움바 스프라이트
    · const float HP_BAR_OFFSET_Y = 0.65f  — 아군 위 Y 오프셋
    · const float HP_BAR_WIDTH    = 0.50f  — 바 너비
    · const float HP_BAR_HEIGHT   = 0.065f — 바 높이

  CreateHpBar() — Awake() 마지막에 호출:
    · 1×1 픽셀 텍스처(MakePixelSprite)로 어두운 배경 SpriteRenderer 생성
      (sortingOrder=30)
    · 동일 방식으로 채움바 SpriteRenderer 생성 (sortingOrder=31)
    · 초기에는 SetActive(false)로 숨김

  UpdateHpBar() — 피해/회복 시 호출:
    · HpRatio < 0.999f && !isDead 일 때만 표시
    · 색상 그라디언트: 초록(100%) → 노랑(50%) → 빨강(0%)
    · 왼쪽 정렬: fillW = HP_BAR_WIDTH * ratio,
      localPosition.x = -HP_BAR_WIDTH/2 + fillW/2

  호출 지점:
    · OnDamaged(): 피해 수신 즉시 갱신
    · SkillRegenCoroutine(): 매 1초 회복 후 갱신 (Cleric 스킬)

■ 설계 결정
  · 아군이 게임 오브젝트 파괴 시 hpBarRoot도 자동 제거 (자식이므로 별도 처리 불필요)
  · hpBarRoot는 transform(root)의 자식 → visualRoot 애니메이션의 영향을 받지 않음
    (체력바가 캐릭터 비틀기/호흡 애니와 독립적으로 안정적으로 표시)
  · 최대 체력 상태에서는 바를 숨김 → 화면 정리, 피해 받은 아군만 시각적 경고
