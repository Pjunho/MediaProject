▶ 발견된 문제: 아군 HP 바 이중 표시 버그

원인 분석
  - 2026-05-04 AllyBase.cs에 체력바 시스템(CreateHpBar / UpdateHpBar)이 구현되었음.
    이 시스템은 Awake() 에서 자동으로 HP 바를 생성하며, HP 최대치일 때 자동으로 숨김.
  - 그러나 AllyPlacer.cs의 DeployAlly() 메서드에는 이전 방식으로 구현된
    AddHpBar() 호출이 그대로 남아 있었음.
  - 결과: 아군 한 명당 HP 바가 두 개(HpBarBg + HpBar) 동시에 생성되어
    같은 위치에 겹쳐 표시되는 시각적 버그 발생.
    기존 AddHpBar() 방식은 항상 표시되므로, 체력이 최대일 때도 바가 보이는
    추가적인 동작 불일치도 존재했음.

2단계 — 신규 기능 구현: 출전 대기열 힌트 HUD 개선
────────────────────────────────────────────────────────────────────────────────

▶ 구현 목표
  기존 힌트 텍스트 "[Space / Click] 아군 출전 (대기: N명)" 은
  대기 인원 수만 표시하고 구체적인 아군 종류를 알 수 없었음.
  다음에 출전할 아군의 이름을 한국어로 표시해 플레이어가 전략적 판단을
  더 빠르게 내릴 수 있도록 개선.

표시 형식
  대기 인원 3명 이상: "[Space / Click]  전사 출전  (다음: 궁수 +2명)"
  대기 인원 2명     : "[Space / Click]  전사 출전  (다음: 궁수)"
  대기 인원 1명     : "[Space / Click]  전사 출전  (마지막)"
  대기열 비어있음   : (텍스트 숨김)

파일 변경 내역

  Assets/AllyPlacer.cs
    - PeekDeployQueue() 메서드 추가
        public AllyType[] PeekDeployQueue() => deployQueue.ToArray();
      Queue<T>.ToArray()는 선입 순서(front→back)로 배열을 반환하므로
      index 0이 바로 다음 출전 아군, index 1이 그 다음 아군.

  Assets/GameManager.cs
    - RefreshDeployHint() 전면 재작성
        AllyPlacer.Instance.PeekDeployQueue() 호출 후
        queue[0](다음), queue[1](그 다음), 나머지 인원 수를 조합하여
        위의 형식에 맞게 힌트 문자열 생성.
    - GetAllyKorName(AllyType t) 정적 헬퍼 추가
        AllyType → 한국어 이름 switch 식 변환
        (전사/궁수/마법사/성직자/도적/성기사)
    - 힌트 UI 배경 너비 370 → 460 픽셀로 확장
    - 힌트 텍스트 영역 너비 360 → 450 픽셀로 확장
      (긴 문자열이 잘리지 않도록)
